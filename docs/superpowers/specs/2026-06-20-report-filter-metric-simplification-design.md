# 报表筛选上提与字段精简设计

日期：2026-06-20

## 1. 背景

现有报表同时展开了新诊、复购、新客转化和历史复购贡献等指标，主列表与“业绩拆解”抽屉的字段过多。新诊/复购、新客/老客本质上是统计范围，而不是每次都必须并列展示的报表列。

本次将这两类拆分上提为筛选条件，并把报表收敛为基础成交指标。用户需要查看某个细分范围在原始总量中的贡献时，再按需展示三个占比字段。

## 2. 目标

- 客户统计范围改为“全部 / 新客 / 老客”。
- 新增“成交类型：全部 / 新诊 / 复购”筛选。
- 报表主列表与“业绩拆解”抽屉固定展示：上报业绩、成交单量、成交客户数、客单价。
- 当客户统计范围或成交类型任一项不是“全部”时，追加业绩占比、单量占比、客户占比。
- 占比分母解除客户统计范围和成交类型限制，保留其余筛选条件与行维度。
- 保持公共筛选栏，仪表盘继续响应这些筛选，但本次不改仪表盘的组件、字段或布局。

## 3. 不做范围

- 不重构仪表盘内容或指标体系。
- 不改变“业绩明细”抽屉中的原始成交记录字段。
- 不增加确认业绩或其他旧指标的替代展示。
- 不接入真实接口、权限、导出或自定义列配置。

## 4. 筛选交互

筛选栏保持公共位置，并在现有“客户统计范围”旁增加“成交类型”。点击“查询”后，两项和时间、咨询师、渠道、项目等条件一起生效。

### 4.1 客户统计范围

| 选项 | 口径 |
| --- | --- |
| 全部 | 不限制客户创建时间 |
| 新客 | 客户创建时间落在当前统计周期内 |
| 老客 | 客户创建时间不落在当前统计周期内 |

“新客 / 老客”按客户创建时间划分，不按首单、是否有历史成交或成交类型划分。

### 4.2 成交类型

| 选项 | 口径 |
| --- | --- |
| 全部 | 不限制成交类型 |
| 新诊 | 仅成交类型为“新诊”的记录 |
| 复购 | 仅成交类型为“复购”的记录 |

两个筛选可以叠加。例如“新客 + 新诊”表示同时满足这两个条件的成交记录。

### 4.3 对仪表盘的影响

由于用户选择保留公共筛选栏，仪表盘仍按所有公共筛选后的记录计算现有汇总值。因此调整客户统计范围或成交类型会改变仪表盘数值；但本次不改变仪表盘的呈现和字段。

## 5. 报表字段

### 5.1 默认字段

主列表和“业绩拆解”抽屉的汇总表默认只展示：

| 字段 | 口径 |
| --- | --- |
| 上报业绩 | 当前行内成交记录的上报金额总和 |
| 成交单量 | 当前行内成交记录数 |
| 成交客户数 | 当前行内去重客户数 |
| 客单价 | 上报业绩 ÷ 成交单量 |

不再展示确认业绩、新诊/复购拆分、新客转化和历史复购贡献等列。

### 5.2 条件字段

只要“客户统计范围”或“成交类型”任一项不是“全部”，主列表和“业绩拆解”抽屉都在基础四列后追加：

| 字段 | 公式 |
| --- | --- |
| 业绩占比 | 当前行上报业绩 ÷ 同行基准上报业绩 |
| 单量占比 | 当前行成交单量 ÷ 同行基准成交单量 |
| 客户占比 | 当前行成交客户数 ÷ 同行基准成交客户数 |

两个筛选都为“全部”时，三个占比列不出现。若分母为 0，展示 `—`，不展示误导性的比例。

## 6. 数据流

每次查询构造两份记录集：

1. **当前记录集**：应用全部公共筛选，包括客户统计范围与成交类型。
2. **基准记录集**：应用相同的时间、组织、渠道、项目、客户池和城市等筛选，但强制将客户统计范围与成交类型设为“全部”。

报表聚合逻辑分别聚合两份记录集，再对同一行匹配并计算占比。

### 6.1 主列表匹配

主列表按当前主维度匹配，例如主维度为咨询师时：

`张敏（新客 + 新诊）的业绩 ÷ 张敏（全部客户 + 全部成交类型）的业绩`

列表仍只展示当前记录集中存在的行；基准只作为行内占比分母，不额外生成行。

### 6.2 业绩拆解匹配

抽屉先锁定主列表的选中行，再按当前拆解维度匹配基准。例如咨询师“张敏”按渠道拆解时：

`张敏 + 信息流（新客 + 新诊）的业绩 ÷ 张敏 + 信息流（全部客户 + 全部成交类型）的业绩`

这样抽屉行与主列表采用同一口径，不会把不同咨询师、渠道或项目的数据混入分母。

### 6.3 明细抽屉

“业绩明细”继续展示当前筛选范围内的原始记录，不增加基准记录，也不增加占比列。

## 7. 实现边界

- 继续复用现有完整指标定义供仪表盘使用；报表主列表和拆解抽屉改用独立的精简字段定义，防止报表删列影响仪表盘。
- 扩展客户统计范围的过滤逻辑以支持老客，并将已有的成交类型数据层筛选接入筛选栏。
- 报表层接收当前记录集与基准记录集；占比只由这两份已聚合结果计算，避免把筛选规则分散到表格组件中。
- 占比列显示条件由当前筛选状态决定，主列表与抽屉复用同一规则。

## 8. 验收标准

- 客户统计范围可选“全部 / 新客 / 老客”，且新客与老客互斥、覆盖全部记录。
- 成交类型可选“全部 / 新诊 / 复购”，并可与客户统计范围组合。
- 公共筛选提交后，仪表盘维持原有展示，但汇总值随公共筛选更新。
- 报表主列表和业绩拆解抽屉默认只展示四个基础字段。
- 任一新筛选不是“全部”时，主列表和业绩拆解抽屉都显示三个占比字段；两项均为“全部”时，三个字段均隐藏。
- 主列表的每个占比分母与当前行主维度一致；抽屉的每个占比分母与当前主维度和拆解维度一致。
- 基准分母保留其余公共筛选，只忽略客户统计范围与成交类型。
- 分母为 0 时展示 `—`。
- 业绩明细抽屉字段保持不变。
