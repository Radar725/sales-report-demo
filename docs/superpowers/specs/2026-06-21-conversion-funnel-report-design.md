# 转化漏斗报表设计

日期：2026-06-21

## 1. 目标与范围

在现有“业绩报表”旁新增“转化漏斗报表”，用于按客户生命周期阶段查看客户量与转化效率。

本次仅新增漏斗报表及其演示数据、筛选、汇总、维度拆解和测试；不改动业绩报表的筛选、指标、列表或金额口径，不接入真实接口、导出、权限或自定义列。

## 2. 页面结构与状态

页面顶部使用两个 Tab：`业绩报表` 与 `转化漏斗报表`。默认展示业绩报表。

两个 Tab 各自保存筛选条件、主维度和抽屉状态：切换到漏斗报表时不改变业绩报表当前查询；切回业绩报表时恢复其原有状态。漏斗报表首次打开的默认值为“新客 + 有效客户”，主维度默认为“咨询师”。

漏斗报表的筛选栏只包含：

| 筛选项 | 选项 / 行为 | 默认值 |
| --- | --- | --- |
| 统计时间 | 日期范围，必填 | 当月截至当天 |
| 客户统计范围 | 全部 / 新客 / 老客 | 新客 |
| 客户类型 | 全部 / 有效客户 / 无效客户 | 有效客户 |
| 咨询师 | 支持按部门与咨询师选择 | 不限 |
| 渠道 | 支持按渠道分类与渠道选择 | 不限 |

主维度只包含：汇总、日期、部门、咨询师、渠道分类、渠道。表格第一列随主维度变化。

## 3. 漏斗客户数据与筛选口径

新增面向漏斗的独立客户事件演示数据。每位客户包含：

- 客户唯一标识、创建时间、客户类型（有效 / 无效）；
- 部门、咨询师、渠道分类、渠道；
- 派单日期、邀约日期、到院日期、成交日期。未发生的阶段使用空值。

演示数据中的阶段保持顺序：派单 → 邀约 → 到院 → 成交。客户类型是独立属性，不由阶段推导。

客户统计范围仅按客户创建时间判断：

| 范围 | 口径 |
| --- | --- |
| 全部 | 不限制创建时间 |
| 新客 | 创建时间落在统计周期内 |
| 老客 | 创建时间不落在统计周期内 |

客户类型与客户统计范围叠加过滤。咨询师和渠道筛选同样只限定客户归属，不改变阶段日期的判定方式。

## 4. 字段与计算口径

表格与维度拆解抽屉均展示以下 12 个字段：

| 字段 | 计算 |
| --- | --- |
| 客户数 | 满足客户筛选与当前维度的去重客户数；不按任何阶段日期限制 |
| 派单客户数 | 派单日期在统计期内的去重客户数 |
| 邀约客户数 | 邀约日期在统计期内的去重客户数 |
| 到院客户数 | 到院日期在统计期内的去重客户数 |
| 成交客户数 | 成交日期在统计期内的去重客户数 |
| 派单率 | 派单客户数 ÷ 客户数 |
| 邀约率 | 邀约客户数 ÷ 客户数 |
| 到院率 | 到院客户数 ÷ 客户数 |
| 成交率 | 成交客户数 ÷ 客户数 |
| 派单邀约率 | 邀约客户数 ÷ 派单客户数 |
| 邀约到院率 | 到院客户数 ÷ 邀约客户数 |
| 到院成交率 | 成交客户数 ÷ 到院客户数 |

所有阶段按各自的发生日期统计，而非按同一批派单客户做队列追踪。因此跨期跟进可能让阶段转化率超过 100%；如实展示计算结果，不截断或重算。任何比率的分母为 0 时展示 `—`。

### 4.1 字段前缀

字段前缀由“客户统计范围 + 客户类型”拼接：

| 客户统计范围 | 客户类型 | 示例字段 |
| --- | --- | --- |
| 新客 | 有效客户 | 新客有效客户数、新客有效派单率 |
| 老客 | 无效客户 | 老客无效客户数、老客无效派单率 |
| 全部 | 有效客户 | 有效客户数、有效派单率 |
| 全部 | 全部 | 客户数、派单率 |

主表与拆解抽屉复用同一份前缀与列定义。

## 5. 维度拆解

每个非“汇总”行提供“维度拆解”操作。点击后打开抽屉，锁定该行的主维度值，再由用户选择允许继续下钻的维度查看同一筛选范围内的漏斗统计。

沿用现有层级规则：不能选择当前维度或汇总；同一层级组仅允许向下钻取，例如部门可拆为咨询师、渠道分类可拆为渠道；不同层级组之间可拆解。汇总行没有可用拆解操作。

## 6. 实现边界与验证

- 在页面容器中增加 Tab 和相互独立的报表状态。
- 漏斗筛选栏独立于业绩报表筛选栏，只复用必要的树形选项构建能力。
- 新建漏斗数据域：客户事件类型、筛选、按维度聚合、比率计算、列名生成。
- 新建漏斗主表和拆解抽屉；不复用业绩报表的金额指标或成交记录聚合。
- 日期为空时阻止查询并显示表单提示；没有匹配记录时显示空表。

测试至少覆盖：客户统计范围和客户类型的组合、字段前缀、五项客户数的各自日期口径、七个比率和零分母、六个主维度聚合、可拆与不可拆维度、两个 Tab 的状态隔离。
