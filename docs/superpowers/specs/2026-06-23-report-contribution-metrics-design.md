# 业绩报表：三项贡献指标

## 目标

在业绩报表主列表与业绩拆解列表中，**新增**三项用于说明当前行在当前筛选结果中结构占比的指标：业绩贡献、成交单量贡献、成交客户贡献。

**与既有「占比」指标并存，不替换。** 占比继续衡量当前行在同维度 baseline 中的份额；贡献衡量当前行在当前筛选结果汇总中的份额。

## 范围

- 包含：业绩报表主列表、业绩拆解列表、对比期展示、指标聚合、字段前缀和自动化测试。
- 不包含：转化漏斗报表、业绩明细抽屉、筛选条件和其他既有指标的排序调整。

## 两套指标定义

### 占比（既有，保留）

| 指标 | 数据字段 | 分子 | 分母 |
| --- | --- | --- | --- |
| 业绩占比 | `reportedAmountRate` | 当前行上报业绩 | 同维度 baseline 行上报业绩 |
| 成交单量占比 | `dealCountRate` | 当前行成交单量 | 同维度 baseline 行成交单量 |
| 成交客户占比 | `customerCountRate` | 当前行成交客户数 | 同维度 baseline 行成交客户数 |

- baseline 记录集：在已应用统计时间与其他筛选后，将客户范围与成交类型放宽为「全部」。
- 主列表：分母为同一主维度值在 baseline 中的汇总。
- 业绩拆解：分母为同一主行、同一拆解维度值在 baseline 中的汇总。
- **显示规则**：仅当客户范围或成交类型非「全部」时展示；「全部 + 全部」时隐藏。

### 贡献（新增）

| 指标 | 数据字段 | 分子 | 分母 |
| --- | --- | --- | --- |
| 业绩贡献 | `reportedAmountContributionRate` | 当前行上报业绩 | 当前筛选条件下的汇总上报业绩 |
| 成交单量贡献 | `dealCountContributionRate` | 当前行成交单量 | 当前筛选条件下的汇总成交单量 |
| 成交客户贡献 | `customerCountContributionRate` | 当前行成交客户数 | 当前筛选条件下的汇总成交客户数 |

- 主列表的分母是当前筛选结果在主维度分组前的汇总值。
- 业绩拆解列表的分母是该拆解表所属主行在拆解维度分组前的汇总值。
- 分母为 0 时，该贡献指标无定义，显示 `—`。
- **显示规则**：无论客户范围和成交类型是否为「全部」，始终展示；「全部 + 全部」时列名不带前缀。

## 展示规则

- 列顺序固定为：
  - `客单价`
  - → `业绩占比 → 成交单量占比 → 成交客户占比`（仅筛选收窄时）
  - → `业绩贡献 → 成交单量贡献 → 成交客户贡献`（始终）
  - → 复购历史占比三列（仅成交类型为复购时，逻辑不变）
- 主列表与业绩拆解列表复用同一组指标定义、顺序和单元格渲染逻辑（`buildReportMetricColumns`）。
- 筛选客户范围或成交类型后，两套指标均沿用现有规则添加前缀。例如：`新客新诊业绩占比`、`新客新诊业绩贡献`。

## 对比期

选择对比时间时，占比与贡献均与其他业绩指标一致：

1. 首行显示本期值。
2. 次行显示相对对比期同指标值的增减幅：`(本期值 - 对比期值) ÷ 对比期值`。

对比期值无定义、为 0，或缺少可匹配的对比行时，次行显示 `—`。

## 实现边界

- 在 `analytics.ts` 中并行计算两套字段：占比走 baseline 分母，贡献走当前筛选汇总分母。
- 在 `reportMetrics.tsx` 中分别定义 `shareMetrics`（占比）与 `contributionMetrics`（贡献），贡献使用独立数据键。
- 主列表与业绩拆解共用列构建与单元格渲染；不维护独立展示逻辑。
- 不改变复购历史占比（`repurchase*TotalRate`）的口径与展示条件。

## 验收与测试

- 筛选收窄时，主列表与拆解列表同时显示占比三列与贡献三列。
- 「全部 + 全部」时仅显示贡献三列，不显示占比三列。
- 占比按 baseline 口径计算；贡献按当前筛选汇总口径计算。
- 两套指标分母为 0 时均显示 `—`。
- 全部筛选下贡献列无前缀；新客/老客和新诊/复购筛选正确添加前缀。
- 选择对比时间时，两套指标均正确显示相对对比期的增减幅；无有效对比值时显示 `—`。
