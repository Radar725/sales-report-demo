# 业绩报表：确认业绩与业绩确认率

## 目标

在业绩报表的主列表和业绩拆解中，新增“确认业绩”和“业绩确认率”两项指标，帮助比较已上报业绩的确认进度。

## 范围

- 包含：业绩报表主列表、业绩拆解、对比期展示、指标聚合和自动化测试。
- 不包含：转化漏斗报表、业绩明细抽屉、既有筛选条件和其他指标重排。

## 计算口径

- 确认业绩：当前维度行所含基础业绩明细的 `confirmedAmount` 直接求和。
- 不按 `dealStatus` 二次过滤。
- 业绩确认率：`确认业绩 ÷ 上报业绩`。
- 当上报业绩为 0 时，业绩确认率无定义，显示 `—`。

## 展示规则

- 指标列固定插入在“上报业绩”之后，顺序为：`上报业绩 → 确认业绩 → 业绩确认率`。
- 主列表与业绩拆解使用同一组指标定义和同一列顺序。
- 业绩明细抽屉不调整；它已有对应的“确认金额”字段。
- 客户范围为新客或老客时，按现有列名规则添加前缀：例如“新客确认业绩”“新客业绩确认率”。其他范围使用无前缀标题。

## 对比期

当已选择对比时间时，两项新增指标与既有业绩指标使用一致的单元格布局：

1. 首行显示本期指标值。
2. 次行显示相对对比期的增减幅：`(本期值 - 对比期值) ÷ 对比期值`。

对比期值为 0、缺少匹配维度，或本期/对比期业绩确认率无定义时，增减幅显示 `—`。

## 实现边界

采用指标定义驱动的最小改动：将两项指标加入业绩报表现有的基础指标、行聚合和对比值集合。主列表与业绩拆解复用这些定义，不维护独立的列或计算逻辑。

## 验收与测试

- 确认业绩直接汇总 `confirmedAmount`，不会受 `dealStatus` 影响。
- 业绩确认率按确认业绩除以上报业绩计算，零分母显示 `—`。
- 主列表和业绩拆解都展示新增两列，且紧跟上报业绩。
- 新客、老客范围正确显示列名前缀。
- 有对比时间时，两项指标正确显示增减幅；无对比时间时保持单行本期值。
