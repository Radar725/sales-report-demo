# 转化漏斗报表：录单 cohort 与当前状态口径设计

日期：2026-06-22

## 1. 目标与范围

将转化漏斗报表从“各转化动作发生时间”统计，调整为“录单时间 cohort + 客户当前生命周期状态”统计。

本次只改转化漏斗报表的数据口径、筛选、字段、演示数据与对应测试。不改业绩报表、页面 Tab、主维度体系、维度拆解规则、真实接口、导出或权限。

## 2. 核心统计口径

### 2.1 录单 cohort

筛选栏中的“统计时间”更名为“录单时间”。其底层字段仍为 `customerCreatedAt`：只有创建时间落在选定日期范围内的客户，才进入该周期的漏斗统计。

当前周期与对比周期分别按各自的录单时间筛选客户。客户类型、部门/咨询师、渠道分类/渠道筛选在录单 cohort 之上叠加。

### 2.2 当前状态

各阶段均读取客户查询时的当前 `status`，不读取派单、邀约、到院或成交动作日期。状态顺序固定如下：

1. 待加微
2. 已加微未派单
3. 派单未邀约
4. 邀约未到院
5. 到院未成交
6. 首次成交
7. 持续复购

阶段客户数按状态累计计算：客户处于某阶段，即计入该阶段和此前所有阶段。例如：

- 已到院客户数 = `到院未成交 + 首次成交 + 持续复购`；
- 已成交客户数 = `首次成交 + 持续复购`；
- 已复购客户数 = `持续复购`。

实现上使用唯一的“状态 → 漏斗层级”映射，不为每个指标单独维护状态集合。

### 2.3 历史数据限制

系统没有客户历史状态快照。因此，某个过往录单 cohort 的数值代表“截至当前”的状态进展；客户后续推进后，旧周期的阶段数和比例也会相应变化。该报表不用于还原任一历史月底的状态快照。

## 3. 筛选与展示

漏斗筛选栏保留：录单时间、对比时间、客户类型、咨询师、渠道。删除“客户统计范围”。

主维度保留：汇总、日期、部门、咨询师、渠道分类、渠道。日期维度按 `customerCreatedAt`（录单时间）分组。

主表及维度拆解抽屉固定展示以下字段，不再显示新客或老客前缀：

| 字段 | 计算 |
| --- | --- |
| 录单客户数 | 筛选后、录单时间落入统计周期的去重客户数 |
| 已加微客户数 | 状态为“已加微未派单”及其后续阶段的客户数 |
| 已派单客户数 | 状态为“派单未邀约”及其后续阶段的客户数 |
| 已邀约客户数 | 状态为“邀约未到院”及其后续阶段的客户数 |
| 已到院客户数 | 状态为“到院未成交”及其后续阶段的客户数 |
| 已成交客户数 | 状态为“首次成交”及其后续阶段的客户数 |
| 已复购客户数 | 状态为“持续复购”的客户数 |
| 加微率 | 已加微客户数 ÷ 录单客户数 |
| 派单率 | 已派单客户数 ÷ 录单客户数 |
| 邀约率 | 已邀约客户数 ÷ 录单客户数 |
| 到院率 | 已到院客户数 ÷ 录单客户数 |
| 成交率 | 已成交客户数 ÷ 录单客户数 |
| 复购率 | 已复购客户数 ÷ 录单客户数 |

任一比率的分母为 0 时展示 `—`。状态累计关系保证每项阶段数不超过录单客户数，且阶段数和阶段率自前向后均不增加。

维度拆解继续锁定主表行的主维度值，并沿用现有可下钻规则；抽屉与主表复用完全相同的字段和计算口径。

## 4. 对比时间

对比期与当前期使用同一计算规则：分别筛选其录单 cohort，再以客户的当前状态计算指标。表格继续展示当前值及相对对比期的变化。

## 5. 数据与实现边界

- 漏斗客户数据以 `customerCreatedAt`、`status`、客户类型和归属维度为核心；旧阶段动作日期不参与本报表计算。
- 删除 `customerScope` 的类型、默认值、控件、过滤逻辑和列名前缀依赖。
- 保持漏斗报表与业绩报表状态独立。
- 不增加历史状态快照、状态变更记录或新的后端接口。

## 6. 验证范围

测试至少覆盖：

1. 每种状态在七项阶段客户数中的累计归属，尤其“到院未成交 + 首次成交 + 持续复购 = 已到院客户数”；
2. 录单时间只决定 cohort 纳入范围，阶段动作日期不影响统计；
3. 六个阶段率、零分母和阶段单调性；
4. 移除客户统计范围后筛选栏不再出现该项，列名不带新客/老客前缀；
5. 主表、维度拆解和对比期使用相同口径。
