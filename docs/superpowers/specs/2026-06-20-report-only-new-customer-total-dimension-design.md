# 报表单页、新客字段与汇总维度设计

日期：2026-06-20

## 1. 目标

将页面收束为报表单页，并在现有报表能力上增加两项范围明确的能力：

- 移除“仪表盘 / 报表”Tab 与仪表盘内容，仅保留报表。
- 当客户统计范围为“新客”时，在列表中展示新客数和新客成交率。
- 主维度增加“汇总”，以不拆分维度的方式返回唯一一行报表数据。

## 2. 页面与交互

筛选栏继续位于页面顶部，筛选条件和查询、重置、导出操作保持不变。筛选栏下直接渲染报表，不再提供 Tab 切换，也不再计算或渲染仪表盘。

主维度下拉框保留现有维度，并新增“汇总”。

| 主维度 | 报表行为 |
| --- | --- |
| 汇总 | 不按任何维度分组，列表固定只有一行，行名为“汇总”。 |
| 现有维度 | 保持当前按所选维度分组的行为。 |

汇总行仍显示两个操作：

- “业绩拆解”按钮可见但置灰，不能打开拆解抽屉。
- “业绩明细”按钮可用，展示当前全部筛选条件下的原始成交记录。

普通维度行的两个操作及抽屉行为不变。

## 3. 新客字段

仅在客户统计范围为“新客”时，主列表和业绩拆解列表在“上报业绩”之后插入以下两列：

| 字段 | 口径 | 分母为 0 |
| --- | --- | --- |
| 新客数 | 统计周期内创建的全部客户数。 | 0 |
| 新客成交率 | 有成交的新客数 ÷ 新客数。 | `—` |

“新客”仍按客户创建时间是否落在统计周期内定义，不按首单、历史成交或成交类型定义。

这两个字段与成交类型筛选解耦：只要客户统计范围为“新客”就显示；成交类型为“全部 / 新诊 / 复购”不改变它们的显示规则。

普通维度行按该行维度范围计算新客数与新客成交率；汇总行按当前筛选范围内的全部数据计算。业绩拆解表在其锁定的主维度范围内，按拆解维度计算相同口径。

## 4. 列顺序与既有规则

主列表和业绩拆解列表的列顺序为：

`主维度（或拆解维度） → 上报业绩 → 新客数 / 新客成交率（仅新客范围） → 成交单量 → 成交客户数 → 客单价 → 条件占比列 → 操作`

既有的条件占比列保持不变：只要客户统计范围或成交类型任一项不是“全部”，就显示业绩占比、单量占比、客户占比。新客字段不影响占比的分子、分母或显示条件。

## 5. 数据流

继续使用两份记录集：

1. 当前记录集：应用全部公共筛选。
2. 基准记录集：保留时间、组织、渠道、项目、客户池、城市等筛选，但将客户统计范围和成交类型强制设为“全部”。

普通维度的汇总、基准占比匹配和抽屉拆解沿用当前逻辑。新增“汇总”维度时，聚合层直接聚合完整当前记录集，返回唯一一行，不按记录字段取值分组；基准记录集也以同样方式生成唯一基准行，用于既有占比计算。

指标层需要显式保留两类不同值：

- 全部新客数：统计周期内创建的客户数，是新客成交率分母。
- 有成交的新客数：当前成交记录涉及的新客去重数，是新客成交率分子。

不能复用同一个字段表示这两个含义。

## 6. 不做范围

- 不保留仪表盘入口或仪表盘组件渲染。
- 不改变公共筛选项、成交类型定义或客户统计范围定义。
- 不改变业绩明细抽屉字段。
- 不改变普通维度的拆解规则。
- 不新增指标自定义、导出逻辑、接口或权限能力。

## 7. 验收标准

- 页面不再出现“仪表盘 / 报表”Tab 或仪表盘内容，筛选栏下直接显示报表。
- 主维度可选“汇总”；选择后列表仅有一行“汇总”。
- 汇总行“业绩拆解”置灰且不可触发；“业绩明细”可查看全部当前筛选结果。
- 客户统计范围为“新客”时，主列表与业绩拆解列表均在上报业绩后显示新客数和新客成交率。
- 客户统计范围不是“新客”时，这两列不显示。
- 新客数为统计周期内创建的全部客户数；新客成交率为有成交新客数除以新客数。
- 新客成交率的分母为 0 时显示 `—`。
- 现有基础指标、条件占比列、普通维度分组和明细字段保持原有行为。
