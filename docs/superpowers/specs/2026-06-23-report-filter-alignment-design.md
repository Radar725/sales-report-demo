# 报表筛选项对齐设计

## 目标

统一业绩报表与转化漏斗报表中相关筛选项的名称、默认值和实际过滤行为：

1. 业绩报表将“城市&机构”更名为“机构”。
2. 转化漏斗报表新增“客户池”筛选。
3. 业绩报表的“客户统计范围”默认选择“全部”。

## 范围

- 包含：两张报表的筛选栏、筛选状态、漏斗演示数据、实际过滤逻辑和自动化测试。
- 不包含：筛选项之外的指标口径、主维度与拆解规则、列布局、导出能力、真实接口。

## 业绩报表

### 机构

- 展开后的筛选标签由“城市&机构”改为“机构”。
- 控件继续使用现有的城市到机构的树形选择：城市为父节点、机构为子节点。
- 现有城市与机构的筛选状态及过滤逻辑不改；标签调整不改变筛选语义。

### 客户统计范围默认值

- 首次进入业绩报表时，“客户统计范围”默认值为“全部”。
- 点击“重置”后，同样恢复为“全部”。
- “全部”表示不按客户创建时间限制；新客和老客的既有定义、筛选效果、指标计算与列名规则均保持不变。

## 转化漏斗报表

### 客户池筛选

- 漏斗客户记录新增 `customerPool` 字段，并为演示数据提供可区分的客户池值。
- 筛选栏新增多选“客户池”；选项基于当前漏斗客户数据去重生成。
- 未选择客户池时不过滤；选择一个或多个客户池时，仅保留所属客户池命中的客户。
- 客户池筛选与录单时间、咨询师、渠道条件取交集。

### 对比与下钻

- 当前期和对比期均使用同一组客户池选择，分别按各自录单时间范围过滤后计算漏斗指标。
- 主表与维度拆解均消费已过滤的客户集合，因此客户池条件在两处一致生效。
- 不调整“录单 cohort + 当前状态”的既有漏斗口径，也不引入历史状态快照。

## 实现边界

- 业绩报表仅改标签与客户统计范围的初始/重置默认值；不删除城市维度，也不改其树形控件。
- 漏斗“客户池”是实际数据过滤条件，而非仅展示控件。
- 不复用业绩报表的客户池选项或记录类型：漏斗独立从自身客户数据生成选项，避免跨报表数据耦合。

## 验收与测试

1. 展开业绩报表筛选后显示“机构”，不再显示“城市&机构”。
2. 业绩报表首次加载和重置后，“客户统计范围”均为“全部”。
3. 漏斗筛选栏显示多选“客户池”，并展示来自漏斗数据的可选值。
4. 指定客户池后，漏斗主表仅统计命中客户；清空选择后恢复全部客户。
5. 启用对比时间和打开维度拆解时，客户池条件仍一致生效。
