# 报表导出二次确认与列表自定义提示设计

## 目标

为业绩报表和转化漏斗报表的导出入口增加二次确认，降低误操作，并在确认前明确 Excel 的数据范围。同步将转化漏斗报表的主维度默认值改为“汇总”，并为两张列表增加列表自定义入口的占位提示。

## 范围

包含：

- 业绩报表的“导出维度数据”和“导出明细数据”。
- 转化漏斗报表的“导出数据”。
- 三个导出入口的确认弹窗、Excel 下载和唯一的导出中 toast。
- 两张主列表“操作”列表头右上角的齿轮按钮及其提示弹窗。
- 转化漏斗报表主维度的默认值。

不包含：

- 真实的列表列配置能力。
- 后端导出、异步任务、权限校验或导出历史。
- 对既有筛选、指标口径、拆解抽屉或报表字段的改动。

## 导出交互

点击任一导出按钮时，先打开确认弹窗。点击取消或关闭弹窗不会下载文件；点击确认后关闭弹窗，展示唯一 toast“导出数据中，请稍等”，并立即在浏览器中下载标准 `.xlsx` 文件。

| 入口 | 确认说明 | Excel 内容 |
| --- | --- | --- |
| 导出维度数据 | 导出当前筛选条件下、按当前选择的主维度（如日期、部门、咨询师）汇总的业绩数据。 | 当前业绩汇总列表的数据，含主维度和指标列，不含操作列。 |
| 导出明细数据 | 导出当前筛选条件下的成交明细数据。 | 当前筛选后的成交记录明细，不含界面操作字段。 |
| 漏斗导出数据 | 导出当前筛选条件下、按当前选择的主维度汇总的转化漏斗数据。 | 当前漏斗汇总列表的数据，含主维度和指标列，不含操作列。 |

导出采用纯前端生成 Excel 文件的方式，不增加服务端接口。文件名应可识别报表类别和导出内容；无需加入未要求的文件命名配置。

## 列表自定义入口

业绩汇总表和漏斗汇总表的“操作”列表头右上角各放置一个正方形齿轮图标按钮。它是列表级入口，不出现在数据行中。

点击按钮后打开一个独立提示弹窗，正文固定为：

> 支持用户自定义列表

本次不提供字段勾选、排序、保存或恢复默认配置能力。

## 默认值

转化漏斗报表的主维度初始值从“咨询师”调整为“汇总”。业绩报表的默认主维度保持不变。

## 实现边界

- 使用一个共享的导出确认弹窗组件，接收标题、说明和确认回调，统一三处交互。
- 使用一个前端导出工具，把业绩汇总、成交明细和漏斗汇总的数据转换为 Excel 工作表并下载。
- 齿轮提示弹窗的开关状态保留在各自表格组件内；不因静态提示而抬升到页面根组件。
- 不修改现有筛选、计算和拆解链路。

## 验收与测试

- 三个导出入口均先显示与入口匹配的确认说明。
- 取消确认不触发下载；确认触发下载并显示“导出数据中，请稍等”。
- 维度导出的说明明确是“按当前选择的主维度汇总”，而不是笼统的筛选结果。
- 漏斗报表首次打开时，主维度显示“汇总”。
- 两张列表的“操作”表头均有齿轮按钮，点击后能看到“支持用户自定义列表”。
- 通过现有自动化测试和生产构建；下载测试使用浏览器下载 API 的替身，不依赖本机真实下载目录。
