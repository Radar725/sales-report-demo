## 1. 需求背景

咨询部是 CRM 系统的核心使用团队，日常经营中需要持续跟踪两类关键问题：**成交结果是否达标**，以及**录单客户在各业务阶段的转化是否顺畅**。

当前业务存在以下痛点：

1. **数据查看分散**：业绩数据与客户转化数据缺乏统一入口，管理者需要在多个模块间切换，难以形成「过程 + 结果」的完整分析视角。
2. **维度分析不足**：组织、渠道、项目、机构等维度的业绩与转化表现难以按统一口径横向对比，无法快速定位差异来源。
3. **对比与下钻能力弱**：缺少灵活的同期对比（如本月 vs 上月）及多维度拆解能力，问题定位依赖人工导出与二次加工。
4. **权限与口径不统一**：不同角色可见的数据范围、统计口径若缺乏明确规则，容易导致「同一指标、不同数字」的协作摩擦。

基于上述背景，需在 CRM 中建设**业绩报表**与**转化漏斗报表**两类分析能力，为咨询部各级角色提供统一、可下钻、可对比的数据分析工具。

## 2. 需求目标

### 2.1 业务目标

| 目标 | 说明 |
| --- | --- |
| 统一业绩分析入口 | 支持按时间、客户范围、成交类型及多业务维度查看成交结果，覆盖金额、单量、客户数及衍生指标。 |
| 统一转化分析入口 | 支持按录单时间圈定客户样本，以漏斗形式展示从录单到复购各阶段的客户数与转化率。 |
| 提升问题定位效率 | 通过主维度切换、维度拆解、业绩明细下钻及同期对比，帮助用户快速发现异常并追溯原因。 |
| 保障数据一致性 | 复用 CRM 现有数据权限，明确统计口径与展示规则，确保同角色、同筛选条件下数据结果一致。 |

### 2.2 用户目标

| 用户角色 | 核心诉求 |
| --- | --- |
| 咨询部负责人、咨询区部部长 | 查看所辖范围的业绩与转化表现，从组织、渠道等维度定位差异。 |
| 咨询师小组长 | 查看本小组业绩与转化，并对成员、渠道等维度继续拆解分析。 |
| 咨询师 | 查看本人数据范围内的业绩与转化情况。 |

### 2.3 成功指标

- 咨询部各级角色可在各自权限范围内独立完成日常业绩复盘与转化分析，无需额外导出加工。
- 两张报表的筛选、汇总、对比、拆解口径与本 PRD 定义一致，验收用例全部通过。
- 切换报表页签时，各自已提交的查询状态互不干扰。

## 3. 产品范围

本产品包含以下两类报表，通过页面顶部页签切换：

| 报表 | 定位 | 基础数据 |
| --- | --- | --- |
| 业绩报表 | 分析成交结果及其结构 | 业绩明细 |
| 转化漏斗报表 | 分析录单客户的阶段转化 | 客户记录 |

**页面结构**：每个页签由查询区、主维度选择区、汇总列表组成；维度拆解以右侧抽屉承载。页面布局与交互细节以当前报表页面实现为准。

**非目标**（本 PRD 不包含）：

- 自定义指标、自由拖拽透视表或三维交叉分析；
- 图表联动、报表保存或个人看板配置；
- 转化漏斗的客户明细下钻。

## 4. 通用规则

以下规则对两张报表均生效：

| 规则 | 说明 |
| --- | --- |
| 筛选状态独立 | 业绩报表与转化漏斗报表各自维护一套筛选条件。切换页签不会覆盖另一张报表已提交的查询状态；主维度选择也相互独立。 |
| 数据范围优先 | 先按当前账号的数据权限取数，再叠加页面筛选条件进行统计（详见各报表「数据范围」章节）。 |
| 树形筛选项 | 所有层级形式的筛选项均按**叶子节点**对应的数据字段过滤；选中上级节点仅表示批量选中其下全部叶子节点，不单独按上级字段过滤。 |
| 层级维度汇总 | 存在父子层级关系的维度，在主维度汇总、维度拆解及对比计算时，须同时展示叶子节点及其全部上级节点的汇总行；上级节点指标为其整个子树范围内数据的汇总。 |
| 空值展示 | 比例类指标分母为 0、缺少对比行或对比期值为 0 时，展示「—」，不以 0 替代。 |

## 5. 业绩报表

### 5.1 功能概述

业绩报表用于分析成交结果。用户可按统计时间、客户范围、成交类型及多业务维度筛选，按主维度汇总查看上报业绩、确认业绩、成交单量等指标，并支持同期对比、维度拆解与业绩明细下钻。

**数据权限**：列表数据取决于当前账号可见的员工范围；员工业绩明细的可见性依据业绩明细的**合作人**字段关联判断。

### 5.2 筛选项

#### 5.2.1 统计时间

| 属性 | 规则 |
| --- | --- |
| 默认值 | 本月 1 日至当天 |
| 是否可清空 | 不可清空 |
| 统计口径 | 按业绩明细的**上报日期**落在统计时间范围内的记录进行统计；定金不计入统计 |

#### 5.2.2 对比时间

| 属性 | 规则 |
| --- | --- |
| 默认值 | 上月 1 日至上月同日；若上月不存在同日（如 3 月 31 日对应 2 月），结束日期取上月最后一天 |
| 是否可清空 | 可清空；清空后列表仅展示本期值，不展示增减幅 |
| 联动规则 | 用户修改统计时间时，对比时间保持原值（包括已清空状态），不自动跟随变化 |
| 重置行为 | 点击「重置」时，对比时间恢复为上述默认值 |

#### 5.2.3 客户统计范围

| 选项 | 含义 |
| --- | --- |
| 全部 | 不限制客户创建时间 |
| 新客 | 客户创建时间落在统计时间范围内 |
| 老客 | 客户创建时间早于统计时间起始日 |

客户统计范围按客户创建时间划分，可与成交类型组合使用；两者为不同维度，交叉筛选时取交集。

#### 5.2.4 成交类型

| 选项 | 含义 |
| --- | --- |
| 全部 | 同时统计新诊与复购成交 |
| 新诊 | 仅统计成交类型为「新诊」的记录 |
| 复购 | 仅统计成交类型为「复购」的记录 |

#### 5.2.5 咨询师

| 属性 | 规则 |
| --- | --- |
| 选项数据范围 | 当前账号数据权限内、状态为启用的咨询师，及其所属部门；以「部门 → 咨询师」树展示 |
| 默认值 | 空（不限） |
| 筛选口径 | 按业绩明细的**咨询师**（合作人）字段过滤 |

#### 5.2.6 渠道

| 属性 | 规则 |
| --- | --- |
| 选项数据范围 | 全部启用渠道及其归属的渠道分类；以「渠道分类 → 渠道」树展示 |
| 默认值 | 空（不限） |
| 筛选口径 | 按业绩明细的**渠道**字段过滤 |

#### 5.2.7 项目

| 属性 | 规则 |
| --- | --- |
| 选项数据范围 | 全部启用项目及其归属的项目分类；以「项目分类 → 项目」树展示 |
| 默认值 | 空（不限） |
| 筛选口径 | 按业绩明细的**项目**字段过滤 |

#### 5.2.8 机构

| 属性 | 规则 |
| --- | --- |
| 选项数据范围 | 全部启用机构及其归属城市；以「城市 → 机构」树展示 |
| 默认值 | 空（不限） |
| 筛选口径 | 按业绩明细的**机构**字段过滤 |

#### 5.2.9 客户池

| 属性 | 规则 |
| --- | --- |
| 选项数据范围 | 全部启用的客户池 |
| 默认值 | 空（不限） |
| 筛选口径 | 按客户身上的**客户池**字段过滤 |

### 5.3 操作按钮

| 按钮 | 说明 |
| --- | --- |
| 查询 | 将当前筛选项提交到报表，刷新列表数据 |
| 重置 | 恢复各筛选项默认值并提交 |
| 导出维度数据 | 导出当前主列表数据：仅包含页面当前选择的**主维度**及其对应指标列的本期值；**不包含**对比期数值与增减幅，**不包含**其他主维度的数据，**不包含**维度拆解抽屉内的数据 |
| 导出明细数据 | 导出当前已提交筛选条件下的全部**业绩明细**记录（与「业绩明细」抽屉所展示的记录范围一致）；**不包含**对比数据 |

### 5.4 主维度

| 属性 | 规则 |
| --- | --- |
| 默认值 | 汇总 |
| 切换方式 | 汇总列表上方的主维度选择器 |

| 主维度 | 含义 | 汇总依据（业绩明细） |
| --- | --- | --- |
| 汇总 | 不分组，展示一行合计 | — |
| 日期 | 按上报日期逐日汇总 | 上报日期 |
| 部门 | 按合作人组织路径汇总，含直属部门及全部上级部门 | 合作人所属部门（层级） |
| 咨询师 | 按合作人（咨询师）汇总 | 合作人（咨询师） |
| 渠道分类 | 按渠道分类路径汇总，含直属分类及全部上级分类 | 渠道归属分类（层级） |
| 渠道 | 按具体渠道汇总 | 渠道 |
| 城市 | 按城市路径汇总，含直属城市及全部上级城市 | 机构所属城市（层级） |
| 机构 | 按成交机构汇总 | 成交机构 |
| 项目分类 | 按项目分类路径汇总，含直属分类及全部上级分类 | 项目归属分类（层级） |
| 项目 | 按成交项目汇总 | 成交项目 |

#### 层级维度汇总规则

树形维度（部门、渠道分类、城市、项目分类）汇总时，除直属节点外，还须展示其**全部上级节点**；上级行指标为其**整个子树**的数据汇总，即使无数据直接挂在该节点下。

示例：所有咨询师均直属于咨询小组，但以部门为主维度统计时，仍须将「咨询一部」「咨询部」「浣花医美」等高层级部门作为独立行展示。

### 5.5 统计列表

#### 5.5.1 通用统计口径

- 统计数据范围：满足页面筛选条件的业绩明细；按主维度或拆解维度分组后，各行仅汇总归属该行的明细。
- **不含定金**：所有业绩相关统计（金额、单量、客户数及由此衍生的比率、占比、贡献等）均排除定金。
- **成交单量**：按成交记录去重计数。一条成交因协作成交可对应多条业绩明细，统计单量时不重复计算。

#### 5.5.2 指标定义

**基础指标（始终展示）**

| 列名 | 统计逻辑 |
| --- | --- |
| 上报业绩 | 归属当前行的业绩明细的上报金额之和 |
| 确认业绩 | 归属当前行的业绩明细的确认金额之和 |
| 业绩确认率 | 确认业绩 ÷ 上报业绩；上报业绩为 0 时显示「—」 |
| 成交单量 | 归属当前行的业绩明细涉及的成交记录去重数 |
| 成交客户数 | 归属当前行的业绩明细涉及的客户去重数 |
| 客单价 | 上报业绩 ÷ 成交单量；成交单量为 0 时按 0 计算 |

**占比指标（条件展示）**

当客户统计范围不为「全部」**或**成交类型不为「全部」时，额外展示以下三列：

| 列名 | 统计逻辑 |
| --- | --- |
| 业绩占比 | 当前行上报业绩 ÷ 同维度下基准行的上报业绩 |
| 成交单量占比 | 当前行成交单量 ÷ 同维度下基准行的成交单量 |
| 成交客户占比 | 当前行成交客户数 ÷ 同维度下基准行的成交客户数 |

基准口径：保留当前全部筛选条件，但将客户统计范围重置为「全部」、成交类型重置为「全部」，再按相同主维度分组后取对应维度值的指标。

**贡献指标（始终展示）**

| 列名 | 统计逻辑 |
| --- | --- |
| 业绩贡献 | 当前行上报业绩 ÷ 当前列表（或拆解范围内）汇总上报业绩 |
| 成交单量贡献 | 当前行成交单量 ÷ 当前列表汇总成交单量 |
| 成交客户贡献 | 当前行成交客户数 ÷ 当前列表汇总成交客户数 |

- 主列表：分母为当前筛选后、按主维度展开前的全部记录汇总值。
- 拆解列表：分母为当前主维度行范围内的全部记录汇总值。

**复购历史占比（条件展示）**

当成交类型选择「复购」时，额外展示以下三列：

| 列名 | 统计逻辑 |
| --- | --- |
| 复购客户历史占比 | 当期复购去重客户数 ÷ 截至统计期结束日的历史累计复购去重客户数 |
| 复购单量历史占比 | 当期复购成交记录去重数 ÷ 截至统计期结束日的历史累计复购成交记录去重数 |
| 复购业绩历史占比 | 当期复购业绩 ÷ 截至统计期结束日的历史累计复购业绩 |

历史数据范围：上报日期 ≤ 统计期结束日、成交类型为「复购」，其余非日期筛选条件（含客户统计范围）仍生效。

**列名动态前缀**

当客户统计范围或成交类型不为「全部」时，相关指标列名自动增加前缀，避免混淆：

| 规则 | 说明 |
| --- | --- |
| 客户范围前缀 | `新客` / `老客` |
| 成交类型前缀 | `新诊` / `复购` |
| 拼接顺序 | 客户范围 + 成交类型 + 指标名（例：`新客新诊上报业绩`） |
| 复购历史占比 | 仅加客户范围前缀（例：`新客复购客户历史占比`） |

#### 5.5.3 表头交互

- **排序**：指标列支持点击排序按钮切换升序 / 降序；主维度列不支持排序。排序依据为**本期值**。
- **口径说明**：指标列标题支持鼠标悬停显示气泡提示，说明该列的计算口径。

#### 5.5.4 默认排序

| 主维度类型 | 默认排序 |
| --- | --- |
| 日期 | 按日期升序排列 |
| 其他维度 | 按上报业绩从高到低排列 |

用户手动排序后保持当前排序，直至切换主维度或重新查询。

### 5.6 列表操作

#### 5.6.1 业绩拆解

| 属性 | 规则 |
| --- | --- |
| 入口 | 非汇总行的「业绩拆解」按钮 |
| 禁用条件 | 主维度为「汇总」时，按钮禁用 |
| 抽屉内容 | 以当前行主维度值 + 当前筛选范围为数据范围，按允许的拆解维度分 Tab 展示与主表相同结构的指标列表；有对比时间时，拆解表同样展示本期值及增减幅 |
| 拆解行操作 | 拆解表每行提供「业绩明细」入口，可查看该行范围内的业绩明细 |

**拆解维度规则**

| 规则 | 说明 |
| --- | --- |
| 汇总行 | 不支持拆解 |
| 同维度 | 不可向自身拆解 |
| 跨分组 | 允许（如咨询师 → 渠道、日期 → 部门） |
| 同分组层级 | 仅允许父 → 子（如部门 → 咨询师、渠道分类 → 渠道）；子级不可反向拆到父级 |

**各主维度可拆解的维度**

| 主维度 | 可拆解维度 |
| --- | --- |
| 汇总 | （无） |
| 日期 | 部门、咨询师、渠道分类、渠道、城市、机构、项目分类、项目 |
| 部门 | 日期、咨询师、渠道分类、渠道、城市、机构、项目分类、项目 |
| 咨询师 | 日期、渠道分类、渠道、城市、机构、项目分类、项目 |
| 渠道分类 | 日期、部门、咨询师、渠道、城市、机构、项目分类、项目 |
| 渠道 | 日期、部门、咨询师、城市、机构、项目分类、项目 |
| 城市 | 日期、部门、咨询师、渠道分类、渠道、机构、项目分类、项目 |
| 机构 | 日期、部门、咨询师、渠道分类、渠道、项目分类、项目 |
| 项目分类 | 日期、部门、咨询师、渠道分类、渠道、城市、机构、项目 |
| 项目 | 日期、部门、咨询师、渠道分类、渠道、城市、机构 |

#### 5.6.2 业绩明细

| 属性 | 规则 |
| --- | --- |
| 入口 | 非汇总行及拆解行的「业绩明细」按钮；汇总行也可点击，查看当前筛选条件下的全部业绩明细 |
| 抽屉内容 | 展示该行对应范围内的业绩明细记录，字段与业绩明细数据一致 |
| 不含对比 | 明细抽屉仅展示本期数据，不展示增减幅 |

### 5.7 数据展示规则

#### 5.7.1 字段格式

| 类型 | 展示规则 |
| --- | --- |
| 金额 | `¥` 前缀 + 千分位格式化（如 `¥12,000`） |
| 整数 | 原样展示 |
| 百分比 | 保留 1 位小数 + `%`（如 `80.0%`） |
| 不可计算 | 显示「—」 |

#### 5.7.2 对比数据展示

选择对比时间并查询后，主列表与拆解列表的指标单元格采用双行布局：

1. **首行**：本期值（格式同上）
2. **次行**：相对对比期的增减幅

| 变化情况 | 展示规则 |
| --- | --- |
| 上涨 | ↑ + 绝对值百分比（如 `↑ 18.6%`） |
| 下降 | ↓ + 绝对值百分比（如 `↓ 5.6%`） |
| 持平 | `0.0%` |
| 不可计算 | `—`（含对比期为 0、对比期无对应行等情形） |

**增减幅公式**：`(本期值 - 对比期值) / 对比期值`，以百分比展示并精确到小数点后 1 位。该规则同样适用于本身为百分比的指标（如业绩确认率 20% 对比 16%，显示 `↑ 25.0%`，而非 4.0 个百分点）。

**对比期取数**：复制当前全部非时间筛选条件，仅将统计时间替换为对比时间范围。新客 / 老客按各自周期的客户创建时间判断。

**维度匹配**

| 维度类型 | 匹配规则 |
| --- | --- |
| 非日期维度 | 按维度值精确匹配（如本期「张晓」对应对比期「张晓」） |
| 日期维度 | 按同一日号匹配（如 6 月 1 日对应 5 月 1 日）；当前月多出的日期无对比期同日时，仅展示本期值，增减幅显示「—」 |

未选择对比时间时，指标单元格仅展示本期值。

### 5.8 数据范围

| 维度 | 规则 |
| --- | --- |
| 基础数据 | 业绩明细记录。一条成交因协作成交可产生多条业绩明细（按合作人拆分），统计时排除定金 |
| 时间范围 | 统计时间按业绩明细的**上报日期**过滤 |
| 权限范围 | 列表数据取决于当前账号可见的员工范围；员工业绩明细的可见性依据业绩明细的**合作人**字段关联判断。筛选、汇总、对比、拆解与明细下钻均只基于该可见范围内的数据 |

## 6. 转化漏斗报表

### 6.1 功能概述

转化漏斗报表用于分析录单客户的阶段转化。用户按录单时间圈定统计样本，查看从录单到复购各阶段的客户数与转化率，并支持同期对比与维度拆解。

**数据权限**：列表数据取决于当前账号可见的**客户**范围。

### 6.2 筛选项

#### 6.2.1 录单时间

| 属性 | 规则 |
| --- | --- |
| 默认值 | 本月 1 日至当天（闭区间） |
| 是否可清空 | 不可清空 |
| 统计口径 | 仅统计客户**创建时间**（录单时间）落在该范围内的客户，作为当期漏斗的统计样本 |

#### 6.2.2 对比时间

| 属性 | 规则 |
| --- | --- |
| 默认值 | 上月 1 日至上月同日；若上月不存在同日，结束日期取上月最后一天 |
| 是否可清空 | 可清空；清空后列表仅展示本期值，不展示增减幅 |
| 联动规则 | 用户修改录单时间时，对比时间保持原值（包括已清空状态），不自动跟随变化 |
| 重置行为 | 点击「重置」时，对比时间恢复为上述默认值 |

#### 6.2.3 咨询师

| 属性 | 规则 |
| --- | --- |
| 选项数据范围 | 当前账号数据权限内、状态为启用的咨询师，及其所属部门；以「部门 → 咨询师」树展示 |
| 默认值 | 空（不限） |
| 筛选口径 | 按客户的**咨询师**字段过滤 |

#### 6.2.4 渠道

| 属性 | 规则 |
| --- | --- |
| 选项数据范围 | 全部启用渠道及其归属的渠道分类；以「渠道分类 → 渠道」树展示 |
| 默认值 | 空（不限） |
| 筛选口径 | 按客户的**渠道**字段过滤 |

#### 6.2.5 客户池

| 属性 | 规则 |
| --- | --- |
| 选项数据范围 | 全部启用的客户池 |
| 默认值 | 空（不限） |
| 筛选口径 | 按客户的**客户池**字段过滤 |

### 6.3 操作按钮

| 按钮 | 说明 |
| --- | --- |
| 查询 | 将当前筛选项提交到报表，刷新列表数据 |
| 重置 | 恢复各筛选项默认值并提交 |
| 导出数据 | 导出当前主列表数据：仅包含页面当前选择的**主维度**及其对应指标列的本期值；**不包含**对比期数值与增减幅，**不包含**其他主维度的数据，**不包含**维度拆解抽屉内的数据 |

### 6.4 主维度

| 属性 | 规则 |
| --- | --- |
| 默认值 | 汇总 |
| 切换方式 | 汇总列表上方的主维度选择器 |

| 主维度 | 含义 | 汇总依据（客户） |
| --- | --- | --- |
| 汇总 | 不分组，展示一行合计 | — |
| 日期 | 按录单日期逐日汇总 | 客户创建时间 |
| 部门 | 按客户组织路径汇总，含直属部门及全部上级部门 | 客户归属部门（层级） |
| 咨询师 | 按客户归属咨询师汇总 | 客户归属咨询师 |
| 渠道分类 | 按渠道分类路径汇总，含直属分类及全部上级分类 | 客户渠道归属分类（层级） |
| 渠道 | 按客户具体渠道汇总 | 客户渠道 |

#### 层级维度汇总规则

与业绩报表一致：涉及部门、渠道分类等树形维度时，须同时展示直属节点及其全部上级节点（如「咨询部」「浣花医美」等顶层数据）。

### 6.5 统计列表

#### 6.5.1 统计样本与阶段口径

- 统计样本为录单时间落在统计期内的客户。
- 各阶段指标读取客户**当前状态**，按累计口径统计：达到某阶段后的客户同时计入该阶段及此前所有阶段。
- 阶段数量及阶段率中，除「录单客户数」外，其余阶段指标仅统计**有效客户**。
- 客户状态进阶顺序：待加微 → 已加微未派单 → 已派单未邀约 → 已邀约未派单 → 已派单未到院 → 已到院未成交 → 已成交未复购 → 已复购

#### 6.5.2 指标定义

**客户数量**

| 列名 | 统计逻辑 |
| --- | --- |
| 录单客户数 | 统计期内全部去重客户数（含有效与无效客户） |
| 有效客户数 | 客户类型为「有效」的去重客户数 |
| 已加微客户数 | 有效客户中，当前状态达到「已加微」及之后阶段的客户数 |
| 已派单客户数 | 有效客户中，当前状态达到「已派单」及之后阶段的客户数 |
| 已邀约客户数 | 有效客户中，当前状态达到「已邀约」及之后阶段的客户数 |
| 已到院客户数 | 有效客户中，当前状态达到「已到院」及之后阶段的客户数 |
| 已成交客户数 | 有效客户中，当前状态达到「已成交」及之后阶段的客户数 |
| 已复购客户数 | 有效客户中，当前状态达到「已复购」阶段的客户数 |

**转化率**

| 列名 | 统计逻辑 |
| --- | --- |
| 有效客户率 | 有效客户数 ÷ 录单客户数 |
| 加微率 | 已加微客户数 ÷ 有效客户数 |
| 派单率 | 已派单客户数 ÷ 有效客户数 |
| 邀约率 | 已邀约客户数 ÷ 有效客户数 |
| 到院率 | 已到院客户数 ÷ 有效客户数 |
| 成交率 | 已成交客户数 ÷ 有效客户数 |
| 复购率 | 已复购客户数 ÷ 有效客户数 |

任一比率的分母为 0 时显示「—」。

#### 6.5.3 表头交互

- **排序**：指标列支持点击排序按钮切换升序 / 降序；排序依据为**本期值**。
- **口径说明**：指标列标题支持鼠标悬停显示气泡提示，说明该列的计算口径。

#### 6.5.4 默认排序

| 主维度类型 | 默认排序 |
| --- | --- |
| 日期 | 按日期升序排列 |
| 其他维度 | 按录单客户数从高到低排列 |

用户手动排序后保持当前排序，直至切换主维度或重新查询。

### 6.6 列表操作

#### 6.6.1 维度拆解

| 属性 | 规则 |
| --- | --- |
| 入口 | 非汇总行的「维度拆解」按钮 |
| 禁用条件 | 主维度为「汇总」时，按钮禁用 |
| 抽屉内容 | 以当前行主维度值 + 当前筛选范围为数据范围，按允许的拆解维度分 Tab 展示与主表相同结构的指标列表；有对比时间时，拆解表同样展示本期值及增减幅 |

**拆解维度规则**

| 规则 | 说明 |
| --- | --- |
| 汇总行 | 不支持拆解 |
| 同维度 | 不可向自身拆解 |
| 跨分组 | 允许（如部门 → 渠道、日期 → 咨询师） |
| 同分组层级 | 仅允许父 → 子（如部门 → 咨询师、渠道分类 → 渠道）；子级不可反向拆到父级 |

**各主维度可拆解的维度**

| 主维度 | 可拆解维度 |
| --- | --- |
| 汇总 | （无） |
| 日期 | 部门、咨询师、渠道分类、渠道 |
| 部门 | 日期、咨询师、渠道分类、渠道 |
| 咨询师 | 日期、渠道分类、渠道 |
| 渠道分类 | 日期、部门、咨询师、渠道 |
| 渠道 | 日期、部门、咨询师 |

### 6.7 数据展示规则

#### 6.7.1 字段格式

| 类型 | 展示规则 |
| --- | --- |
| 整数（客户数） | 原样展示，无千分位、无货币符号 |
| 百分比（转化率） | 保留 1 位小数 + `%`（如 `50.0%`） |
| 不可计算 | 显示「—」 |

#### 6.7.2 对比数据展示

选择对比时间并查询后，主列表与拆解列表的指标单元格采用双行布局，规则与业绩报表一致：

1. **首行**：本期值
2. **次行**：相对对比期的增减幅（↑ / ↓ / 持平 / —）

**增减幅公式**：`(本期值 - 对比期值) / 对比期值`，精确到小数点后 1 位。

**对比期取数**：复制当前全部非时间筛选条件，仅将录单时间替换为对比时间范围；对比期客户同样按录单时间圈定的客户群计算。

**维度匹配**

| 维度类型 | 匹配规则 |
| --- | --- |
| 非日期维度 | 按维度值精确匹配 |
| 日期维度 | 按同一日号匹配；当前月多出的日期无对比期同日时，增减幅显示「—」 |

未选择对比时间时，指标单元格仅展示本期值。

### 6.8 数据范围

| 维度 | 规则 |
| --- | --- |
| 基础数据 | 客户记录 |
| 时间范围 | 录单时间按客户的**创建时间**过滤，确定当期统计样本 |
| 权限范围 | 列表数据取决于当前账号可见的**客户**范围。筛选、汇总、对比与拆解均只基于该可见范围内的客户 |

## 7. 验收标准

| 序号 | 验收项 |

| 1 | 登录账号在两张报表中只能看到自身数据范围内的业绩明细或客户数据，且该范围在筛选、汇总、对比、拆解和下钻中一致生效。 |
